# napari-actin-density

[![napari hub](https://img.shields.io/endpoint?url=https://api.napari-hub.org/shields/napari-actin-density)](https://napari-hub.org/plugins/napari-actin-density)
[![PyPI](https://img.shields.io/pypi/v/napari-actin-density.svg)](https://pypi.org/project/napari-actin-density)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

A [napari](https://napari.org) plugin for quantifying **cortical actin density** from cryo-electron tomography (cryo-ET) data.

Given a membrane surface mesh (`.vtp`) and a binary actin segmentation volume (`.mrc`), this plugin computes the local actin filament density at each vertex of the membrane surface, expressed as **µm of filament per µm² of membrane**.

---

## Overview

This plugin implements a cortical actin density pipeline designed for cryo-ET data:

1. **Load mesh** — reads a triangulated surface mesh in `.vtp` format (e.g. generated by [surface-morphometrics](https://github.com/GrotjahnLab/surface_morphometrics))
2. **Skeletonize actin** — thins the binary actin mask to a 1-voxel-wide skeleton
3. **Clean skeleton** — removes small spurious components below a size threshold
4. **Orient normals** — computes and orients surface normals inward (toward the cytoplasm)
5. **Compute density** — for each mesh vertex, counts actin skeleton points within an oriented cortical cylinder (defined by a depth and lateral radius) and converts to µm/µm²
6. **Save results** — exports a density-mapped `.vtp`, a per-vertex `.csv`, and a `.json` with parameters and summary statistics

Results are visualized directly in Napari as a turbo colormap heatmap on the membrane surface.

## Usage

1. Open Napari
2. Go to **Plugins → Cortical Actin Density**
3. Select your input files and set parameters
4. Click **▶ Run Pipeline**

### Input files

| Field | Description |
|-------|-------------|
| Mesh (.vtp) | Triangulated surface mesh of the membrane |
| Actin mask (.mrc) | Binary segmentation of actin filaments (same coordinate space as the tomogram) |
| Output folder | Directory where results will be saved |

### Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| Voxel size | 12.4 Å | Pixel size of the tomogram in Ångströms |
| Min skeleton component | 20 voxels | Minimum component size to keep after skeletonization (removes noise) |
| Cortex depth | 200 nm | Depth of the cortical search cylinder along the surface normal |
| Lateral radius | 100 nm | Lateral radius of the search cylinder |

### Output files

| File | Description |
|------|-------------|
| `cortical_actin_density.vtp` | Mesh with density as point data — open in ChimeraX or ParaView |
| `cortical_actin_density.csv` | Per-vertex table with coordinates, area, and density |
| `cortical_actin_density_params.json` | Pipeline parameters and summary statistics |

## Method

For each mesh vertex **v** with inward normal **n**, the plugin counts actin skeleton points inside an oriented cylinder:

- Points must lie within `cortex_depth` nm along **n** (inward from the surface)
- Points must lie within `lateral_radius` nm perpendicular to **n**

The local density is then:

```
density (µm/µm²) = (actin_points × voxel_size_nm × 1e-3) / (π × lateral_radius_nm² × 1e-6)
```

Normal orientation is handled automatically:
- **2-component meshes** (e.g. inner + outer membrane): normals point away from the opposing membrane
- **1-component meshes**: normals point toward the actin centroid (or mesh centroid if no actin is present)

---

## Compatibility

Meshes generated by [surface-morphometrics](https://github.com/GrotjahnLab/surface_morphometrics) are directly compatible. The mesh coordinates should be in Ångströms and the actin mask should be in the same voxel space as the original tomogram.
